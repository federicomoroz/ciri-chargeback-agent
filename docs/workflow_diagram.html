DECISIONES DE DISEÑO
  • n8n orquesta · FastAPI ejecuta · lógica nunca en n8n
  • QueryBuilder determinístico · sin LLM para queries RAG
  • Políticas como datos en Qdrant · editables vía API sin redeploy
  • Guardrail: APPROVE + BLOCKER activo → flag alucinación
  • Caché semántico threshold 0.92 · reduce costo LLM
  • Prompts versionados (v1_*) · trazabilidad de cambios
RAG PIPELINE
  • Modelo: paraphrase-multilingual-MiniLM-L12-v2
  • 384 dims · ONNX Runtime (fastembed) · ~80MB RAM
  • Inicialización lazy · modelo carga en primer request
  • Políticas: top-17 · threshold 0.0 · LLM filtra relevancia
  • Casos similares: top-5 · threshold 0.40 · semántico
  • SQLite + Qdrant unidos en contexto del LLM (hechos + semántica)
DEPLOYMENT
  • API: Render.com · Docker · python:3.11-slim
  • Vectores: Qdrant Cloud · sa-east-1 · 17 políticas + 60 casos
  • Orquestación: n8n Cloud · workflow activo · 23 nodos
  • LLM: Anthropic claude-haiku-4-5-20251001
  • Observabilidad: Langfuse (opcional) · NoOpTracer en tests
  • Tests: 50/50 · pytest · SQLite in-memory · mocks LLM
